   2. 설치시 주의사항

      주의 사항이란건 없습니다. 단, DB 접속용 정보 파일인 dbconn_information.php3
      파일의 내용을 사용하실 분의 MySQL Host,id,pass로 꼭 수정하신뒤에 사용하시길
      당부드립니다. 
      그리고 본 프로그램에서는 회원가입시 주소 자동 입력기능을 사용합니다. 그러니
      첨부되는 우편번호(주소록) txt 파일을 zipcode 라는 Table을 만드셔야 됩니다.

      만드는 법

        1. zipcode.txt 파일을 계정으로 UPLoad 합니다. 그리고 Telnet 로 계정접속을
           하신뒤에 MySQL 에 먼저 접속하신후 다음과 같은 Table 을 먼저 생성합니다.

           CREATE TABLE zipcode (code CHAR(7) NOT NULL, city VARCHAR(20) NOT NULL,
           dong VARCHAR(60) NOT NULL);

           또는 

           http://사용자주소/calendar/dbsql/zipcode_sql.php3 로 접속하여 생성 ^^

        2. 이상없이 Table 이 만들어 졌다면, 우편번호 데이터를 Table 에 삽입합니다.

           LOAD DATA INFILE '/파일경로/zipcode.txt' REPLACE INTO TABLE zipcode 
           FIELDS TERMINATED BY '|';

           zipcode.txt 파일의 경로는 zipcode.txt 파일이 있는 폴더에서 pwd 라고 입
           력하면 경로가 화면에 표시됩니다.
           표시된경로/zipcode.txt 라고 입력하시면 됩니다.

           이상없이 성공했다면 아래와 같은 메시지를 나타냅니다.

           Query OK, 8785 rows affected (0.05 sec)
           레코드: 8785개  삭제: 0개  스킵: 0개  경고: 0개

        3. 1,2 번에서 2줄로 되어있습니다만 원래는 1줄입니다. 연결해서 주~욱 적으시
           면 됩니다.

        4. 위의 진행 과정에서 에러가 없었다면, 이상없이 잘 설치된 겁니다. 아래와
           같이 테스트 해보세요. 

           SELECT * FROM zipcode LIMIT 10;

      관리자 모드로 접속하실때에 주의사항

         1. 관리자 모드를 지원합니다. 관리자 id와 password는 admin_information.php3
            라는 파일안에 저장됩니다. DB를 이용하지 않고 File 입출력으로 처리한 거
            죠.
            본 프로그램을 설치하시고 관리자모드로 접속하시면 초기설정으로 다음과 같
            이 설정됩니다.(admin_information.php3 파일 내용을 보시면 금방 이해 되실
            듯... ^^)

            아 이 디 = master
            비밀번호 = 0000

            암호는 crypt() 함수를 이용해 암호화 되서 저장됩니다. 그리고 만일 사용하
            시던중에 암호를 잊어버리셨다면, 과감하게 admin_information.php3 파일을 
            제거하시면 초기 설정으로 돌아갑니다.

         2. admin 폴더의 퍼미션을 777로 설정해 주세요. 그래야 이상없이 admin_informa
            tion.php3 파일이 생성됩니다. 후에 관리자 모드에서 id, password 의 값을
            변경할려 할때도 퍼미션을 777로 해놓으셔야 이상없이 변경이 되겠죠? ^^


   3. 사용하기

      프로그램과 관련된 모든 파일을 계정으로 업로드 합니다. 예를 들자면
      public_html 안에 calendar 폴더를 만들고, 그 안에다 모든 파일들을 업로드 합니
      다. public_html 은 보통 기본으로 생성되는 루트 폴더죠 ^^a 상황에 맞게 올리시
      면 됩니다.
      그리고, 위에서도 언급했듯이 dbconn_information.php3 파일의 내용을 수정합니다
      업로드 하기전에 먼저 수정해서 올리셔도 상관 없습니다. 내용을 정확하게 수정만
      하시면 됩니다.
      그런다음에 웹브라우져로 http://사용자주소/calendar/dbsql/calendarinfo_sql.php3
      로 접속합니다. dbcon_information.php3 파일내의 정보가 정확하다면 MySQL DB 안
      에 wscalendar_info 라는 Table 이 생성됩니다. 이 Table 은 사용자 관리 Table 
      입니다.
      그리고 Telnet 으로 접속하신 후에 calendarinfo_sql.php3 파일의 퍼미션을 000으
      로 변경해 주시던지, 삭제하시면 됩니다.

      모든 작업이 끝났다면 http://사용자주소/calendar/calendar.php3 로 접속하시면
      이쁜 달력이 화면에 나타납니다. 일정관리를 사용하시려면 왼쪽 상단의 메뉴들 중
      에서 사용자등록 아이콘을 누르고 사용자 등록을 하신후에 로그인 아이콘을 누르
      고 접속하신후 일정관리를 사용하시면 됩니다.
      그 외 메뉴는 좀만 살펴보시면 바로 이해 되실듯 싶네요 --..
