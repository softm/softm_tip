------------ 본문
1. 변수 (Variables) 란?
    가. 변경될수 있는 값.
    나. 프로그램 동작을 위해 필요한 데이터들의 주소 공간을
       인간이 알 수 있는 이름으로 명명한것.

    * 참고

2. 변수명 선언 규칙
    가. 변수 이름은 256자 이상 길어질 수 없다.
    나. 변수 이름에서 사용할 수 있는 문자는 영문자, 숫자, 
        밑줄 문자(_) 뿐이다.
    다. 변수 이름에서 첫 번째 문자는 반드시 영문자 이어야 한다.
    라. 변수 이름은 Sub, Beep, End 와 같이 베이직 언어에 내장된 
        예약어는 사용할 수 없다.

     * 참고


3. 변수의 명시적 선언과 묵시적 선언 이란?
    가. 명시적 선언
        1. 변수를 선언하고 변수를 사용합니다.
        2. option explicit 키워드를 모듈 상단에 위치시키면
           반드시 명시적인 선언을 사용해야한다.

    나. 묵시적 선언
        1. 변수의 선언 없이 변수를 사용합니다.
        2. 할당되는 값의 형에 따라 자동적으로 데이터 형이 바뀌게된다.

    다. 두 방법의 비교
명시적 선언을 이용하여 변수를 선언하는 것이 좋습니다.
        1. 명시적 선언의 장점
            . 프로그램 오류를 줄일 수 있다.
            . 메모리 사용이 효율적이다.
        2. 암시적 선언의 단점
            . 프로그램 오류가 발생할 가능성이 높다.
            . 메모리 사용에 비효율적이다.
              ( 형태에대한 선언이 없음으로 내부적으로 추가적인 처리부분이 뒤따름 )
4. 변수의 사용범위에 따른 분류
분류	유효범위	Key Word
지역 변수	프로시저안에서만 유효한	Dim, Static
모듈 변수	모듈파일안에서 공유할 수 있는 변수	Dim, Private
전역 변수	프로그램 전체에서 공유하는 변수	Public

    가. 지역 변수
정의	프로시저 내에서만 사용가능한 변수
선언 위치	프로시저 내부
선언 형식	[ Dim | Stactic ] 변수명 [As 데이터형태명 ]
할당 형식	변수명 = 값
1. Dim을 이용한 지역변수 선언
가. 특징
. 프로시저가 종료되면 내부 변수및 값은 소멸됩니다.


나. 데이터형 지정을 통한 변수 선언 방법 [명시적 선언]
        Sub Test( )
            Dim a As Integer ' <<== 선언
a = 5            ' <<== 할당
End Sub
다. 데이터형 지정없이 통한 변수 선언 방법 [명시적 선언]
    	Sub Test( )
            Dim a ' <<== 선언
            a = 5 ' <<== 할당
End Sub
라. 변수 선언 없이 값 할당 방법  [암시적 선언]
Sub Test( )
            a = 5 ' <<== 할당
        End Sub
